# 대구 버스 정보

## 실기기 APK 인수 테스트

테스트에는 공공데이터포털에서 발급받은 `대구광역시_대구버스정보시스템` 일반 인증키의 **Decoding 값**과 Android 실기기가 필요하다. 이 키를 명령줄, 스크린샷, 이슈, 로그 또는 Git에 남기지 않는다.

1. Android 기기에서 기존 앱을 제거하거나 앱 데이터를 삭제해 저장된 키와 목록을 초기화한다.
2. USB 디버깅이 연결된 기기에 debug APK를 설치한다.

   ```powershell
   adb install -r app\build\outputs\apk\debug\app-debug.apk
   ```

3. 앱을 열고 `공공데이터 API 키 입력` 화면이 나타나는지 확인한다.
4. `API 키`에 일반 인증키의 **Decoding 값**을 입력하고 `키 저장하고 조회`를 누른다.
5. 814번 운행 시간에 `814번 실시간 차량` 목록이 표시되고, 상단에 운행 대수와 `최신 · N초 전` 상태가 보이는지 확인한다. 운행 종료 시간의 `현재 운행 차량 없음`은 정상적인 빈 결과이므로, 목록 검증은 운행 시간에 다시 수행한다.
6. `새로고침`을 누른다. 버튼이 잠시 `조회 중`으로 바뀐 뒤 목록과 경과 시간이 갱신되는지 확인한다.
7. 정상 목록이 보이는 상태에서 기기의 Wi-Fi와 모바일 데이터를 모두 끈 다음 `새로고침`을 누른다. 기존 목록이 사라지지 않고 `네트워크에 연결할 수 없습니다. 마지막 정상 데이터는 유지됩니다.` 배너가 표시되는지 확인한다.
8. 네트워크를 다시 켠 뒤 `API 키 변경`을 누르고, 일부러 잘못된 키를 입력해 `키 저장하고 조회`를 누른다. `API 키를 확인할 수 없습니다. 키를 다시 입력해 주세요.` 오류가 표시되는지 확인한다.
9. 같은 입력 화면에 올바른 Decoding 키를 다시 입력하고 `키 저장하고 조회`를 눌러 814번 목록이 복구되는지 확인한다.

현재 검증 범위는 API 키 입력, 814번 실시간 차량 목록, 수동 새로고침, 마지막 정상 목록 유지 및 오류 분류까지다. **지도와 홈 화면 위젯은 아직 포함되지 않는다.**

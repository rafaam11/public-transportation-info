# 정류장 중심 대구 버스 앱 개편 설계

- 작성일: 2026-07-18
- 상태: 사용자 승인 완료
- 릴리스: 0.7.0 단일 전환

## 목표

출근·퇴근 두 슬롯에 고정된 앱을 최대 20개의 즐겨찾기 정류장 중심 앱으로 전환한다. 홈에서 버스·정류장·장소를 검색하고 현재 위치 또는 장소 주변 정류장을 찾을 수 있어야 한다. 정류장 상세 지도는 첫 프레임부터 정류장을 중심에 두고, 그 정류장에 앞으로 도착할 모든 노선·방향의 초정밀 차량만 동시에 표시한다.

## 화면

- 하단 탭이 없는 단일 홈: `햄버거 | 버스·정류장·장소 검색 | 내 주변`.
- 홈은 정렬 가능한 즐겨찾기 정류장 카드 최대 20개를 캐시 우선으로 표시한다. 화면에 보이는 카드만 네트워크로 갱신한다.
- 검색 결과는 버스·정류장·장소 섹션을 한 화면에 표시한다.
- 내 주변은 사용자가 눌렀을 때만 위치 권한을 요청한다. 500m 안에서 최대 10개를 찾고, 5개 미만이면 1km까지 확장한다.
- 정류장 상세는 전체 노선 도착정보와 접근 중인 초정밀 차량을 함께 보여준다. 가장 빨리 오는 노선을 기본 강조한다.
- 햄버거 메뉴는 즐겨찾기 순서, API 키, 위치 권한, 캐시 갱신, 오늘 API 호출량, 진단, GitHub Releases 공지·업데이트를 제공한다.

## 데이터와 네트워크

- Room v4는 즐겨찾기 정류장, 선택 고정 노선, 정류장 도착 캐시, 위젯 바인딩, 시내 정류장 카탈로그를 저장한다.
- 기존 출퇴근 즐겨찾기·도착 캐시·위젯 슬롯 연결은 삭제한다. API 키와 노선·경로·차량 캐시는 보존한다.
- `getBasic02`에서 시내 정류장을 파싱하고, `getRealtime02?bsId=` 한 번으로 정류장 전체 노선 도착정보를 받는다.
- 초정밀 세션은 노선+방향별로 분리한다. 명단의 현재 순번이 목표 정류장 순번 이하인 차량만 상세 좌표를 요청한다. 강조 노선 3초, 나머지 8초, 명단 30초, 상세 동시성 4개다.
- 위치는 15초까지 정상, 15~30초 지연, 30초 초과 숨김이며 예측·보간하지 않는다.
- 장소 검색은 Cloudflare Worker `GET /v1/places?q=`를 사용한다. Naver 비밀키는 Worker secrets에만 둔다. 대구 결과 최대 5개, 10분 캐시, IP당 분당 30회 제한이다.

## 지도와 위젯

- `MapView` 생성 옵션에 정류장 좌표를 전달해 서울시청 기본 카메라가 한 프레임도 보이지 않게 한다.
- 데이터 도착 후 카메라를 자동으로 덮어쓰지 않는다. 차량 선택과 `노선 전체 보기`만 명시적으로 이동한다.
- 위젯 하나는 즐겨찾기 정류장 UUID 하나에 연결한다. 작은 크기는 2개, 넓은 크기는 4개 노선을 표시한다.
- 위젯 바인딩과 도착 캐시를 Room Flow로 관찰한다. 설정 Activity는 바인딩 저장 후 위젯 ID를 포함한 `RESULT_OK`, 즉시 업데이트, 1회성 부트스트랩 Worker를 수행한다.
- 위젯 네트워크 요청은 수동 새로고침과 앱 동기화에만 발생한다.

## 완료 기준

1. 지도 첫 프레임이 선택 정류장이고 서울시청 플래시가 없다.
2. 홈·검색·내 주변·정류장 상세이 정류장 중심 모델로 동작한다.
3. 선택 정류장에 접근 중인 모든 노선·방향 차량만 지도에 보인다.
4. 캐시와 데이터 시각을 유지하며 부분 장애를 숨기지 않는다.
5. 삼성 Fold에서 위젯 설정 직후 수동 앱 새로고침 없이 데이터가 보인다.
6. 기존 출퇴근 데이터 초기화와 API 키·정적 캐시 보존이 마이그레이션 테스트로 증명된다.

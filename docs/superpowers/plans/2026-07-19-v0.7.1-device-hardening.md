# v0.7.1 Device Hardening Implementation Plan

> **Execution:** Run this plan directly on the user-approved `main` branch. Keep every credential value out of source control and logs.

**Goal:** Ship v0.7.1 with working Naver place search in local/release builds, tappable edge-to-edge controls, and a bounded, honest nearby-location flow.

**Architecture:** Keep the existing Compose/ViewModel structure. Put provider ordering and timeout behavior in a small testable location resolver, expose nearby loading/error states through `StopHomeUiState`, and inject the non-secret Worker base URL through local properties and the release workflow.

**Tech stack:** Kotlin, Jetpack Compose, coroutines, Android `LocationManager`, Gradle, GitHub Actions, Cloudflare Worker.
